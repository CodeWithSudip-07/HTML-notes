🌐 HOW Does Internet Work

🔰 IP Address

  1. Definition:

    An IP (Internet Protocol) address is a unique numerical label assigned to each device connected to a network that uses the Internet Protocol for communication.

  2. Purpose:

    • Identifies a device on the network.
    • Provides the device's location in the network.

  3. Format:

    • Written as numbers separated by dots (IPv4) or colons (IPv6).

🔷 Public IP Address

  1. Definition:

    A Public IP address is the address assigned to a device that is directly accessible over the internet.

  2. Features:

    • Globally unique.
    • Provided by Internet Service Providers (ISP).
    • Used to communicate outside the private/local network.

  3. Example: 103.21.244.0

🔷 Private IP Address

  1. Definition:

    A Private IP address is used within a local/private network (LAN). It is not accessible directly from the internet.

  2. Features:

    • Used for communication within home, office, or internal networks.
    • Devices with private IP connect to the internet through NAT (Network Address Translation).
    • Same private IP ranges can be reused in different networks.

  3. Private IP ranges (IPv4):

    • 10.0.0.0 - 10.255.255.255
    • 172.16.0.0 - 172.31.255.255
    • 192.168.0.0 - 192.168.255.255

  4. Example: 192.168.1.1

🔰 IPv6 (Internet Protocol version 6)

  1. Definition:

    IPv6 is the most recent version of Internet Protocol designed to replace IPv4, which is running out of unique addresses.

  2. Reasons for IPv6:

    • IPv4 has about 4.3 billion unique addresses (limited).
    • IPv6 supports 340 undecillion (3.4 x 10³⁸) unique addresses.

  3. Features of IPv6:

    • 128-bit address length.
    • Written in hexadecimal format, separated by colons.
    • More efficient routing and network auto-configuration.
    • Better support for mobile devices and IoT.

🔷 How Does an IPv6 Address Look Like?

  1. Format:8 groups of 4 hexadecimal digits, separated by colons (:).

  2. Example:

    • Full form: 2001:0db8:0000:0000:0000:ff00:0042:8329
    • Compressed form (leading zeros omitted): 2001:db8::ff00:42:8329

🔰 MAC Address

A MAC Address (Media Access Control Address) is a unique identifier assigned to a device's network interface card (NIC).

  • Format:
    • 48-bit (6-byte) number.
    • Written in hexadecimal (0-9, A-F).
    • Separated by colons (:) or hyphens (-).
    • Example: 00:1A:2B:3C:4D:5E
  • Purpose:
    • Identifies a device physically on a local network (LAN).
    • Works at the Data Link Layer (Layer 2) of the OSI model.
    • Used for communication inside a network segment.

🔰 Port Numbers

  • What is a Port Number?

    • A Port Number is a numerical identifier used in networking to distinguish different processes or services on a single device.

    • Works at the Transport Layer (Layer 4) of the OSI model.
    • Allows multiple services (e.g., web, email, FTP) to run on the same device using the same IP address.
    • Format: 16-bit unsigned integer, range 0 to 65535.

  • Types of Port Numbers

    1. Well-Known Ports (Range: 0 - 1023)

      2. Assigned and controlled by IANA (Internet Assigned Numbers Authority). Used by widely recognized services and protocols.

      • Port 20, 21 → FTP (File Transfer Protocol)
      • Port 22 → SSH (Secure Shell)
      • Port 23 → Telnet
      • Port 25 → SMTP (Simple Mail Transfer Protocol)
      • Port 53 → DNS (Domain Name System)
      • Port 80 → HTTP (Web)
      • Port 443 → HTTPS (Secure Web)

    2. Registered Ports (Range: 1024 - 49151)

      3. Also maintained by IANA. Used by software applications that are not as common as well-known services.

      • Port 1433 → Microsoft SQL Server
      • Port 3306 → MySQL Database
      • Port 3389 → Remote Desktop Protocol (RDP)

    3. Dynamic / Private / Ephemeral Ports (Range: 49152 - 65535)

      4. Not registered with IANA. Used temporarily by client devices when connecting to servers. Assigned automatically by the operating system.

      • Released after the connection ends.
      • Example: A browser may use port 50234 (ephemeral) to connect to a server on port 80 (HTTP) or 443 (HTTPS).

🔰 DNS (Domain Name System) Notes

Topic Description
Definition DNS (Domain Name System) is the “phonebook of the internet,” which translates domain names into IP addresses.
Purpose Converts human-readable names (e.g., google.com) into machine-readable IP addresses (e.g., 142.250.183.78).
How it Works User enters a domain → DNS resolver checks cache → Queries Root Server → TLD Server → Authoritative Server → Returns IP to browser.
Components
  • DNS Resolver: Starts the query process.
  • Root Servers: First step in DNS hierarchy.
  • TLD Servers: Handle domain extensions like .com, .org.
  • Authoritative Servers: Provide the final DNS records.
Types of DNS Records
  • A Record: Domain → IPv4 address.
  • AAAA Record: Domain → IPv6 address.
  • CNAME Record: Alias for another domain.
  • MX Record: Email server information.
  • NS Record: Authoritative name servers.
Importance
  • Makes browsing easier for users.
  • Essential for the scalability of the internet.
  • Supports websites, email delivery, and other online services.
