☯ File Path system:-

  • Relative File Path: Points to a file in relation to the current HTML page's location. Examples include files in the same folder, subfolders, or parent folders.

    1. Example Same Folder: <img src="picture.jpg"> (image is in the same folder as HTML).

    2. Example Subfolder: <img src="images/picture.jpg"> (image is in images subfolder).

    3. Example Parent Folder: <img src="../picture.jpg"> (image is one level up from the current folder).

  • Absolute File Path: Uses the full URL or complete address to point directly to a resource, regardless of the HTML file location.

    1. Example: <img src="https://example.com/images/picture.jpg"> (image is hosted online at the specified address).

🠊 Explanation of Each Element

  1. <!DOCTYPE html>

    • Declares the type of document.
    • Tells the browser to render the page using HTML5 standards.
    • Must be the first line in an HTML document.

  2. <html lang="en"> ... </html>

    • The root element of the HTML document.
    • All other elements are written inside it.
    • lang="en" specifies the language of the page (English in this case).

  3. <head> ... </head>

    • Contains metadata (information about the document).
    • Metadata is not displayed directly on the webpage.
    • Includes title, character encoding, viewport settings, styles, and links.

      • Inside <head>:

    • <meta charset="UTF-8">

      Defines the character encoding (UTF-8 supports most world languages and symbols).

    • <meta name="viewport" content="width=device-width, initial-scale=1.0">

      Ensures the webpage is responsive on different devices.

      1. width=device-width → matches the screen width.
      2. initial-scale=1.0 → sets the initial zoom level.

    • <title>Document</title>

      The title of the webpage.

      1. Appears in the browser's tab.
      2. Important for SEO (search engine optimization).

  4. <body> ... </body>

    • Contains the main visible content of the webpage.
    • Anything written inside <body> is displayed in the browser
    • Examples: headings, paragraphs, images, links, forms, etc.

The boilerplate ensures a standard starting point for every webpage and is expanded as per project requirements (e.g., adding CSS, JavaScript, external libraries).
